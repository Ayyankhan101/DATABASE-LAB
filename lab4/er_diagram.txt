================================================================================
                         RELATIONSHIP SUMMARY
================================================================================

1. CUSTOMERS → ORDERS (One-to-Many)
   - One customer can place many orders
   - Each order belongs to exactly one customer
   - Foreign key: orders.customer_id → customers.customer_id

2. ORDERS → ORDER_ITEMS (One-to-Many)
   - One order can contain many items
   - Each order_item belongs to exactly one order
   - Foreign key: order_items.order_id → orders.order_id

3. PRODUCTS → ORDER_ITEMS (One-to-Many)
   - One product can appear in many order_items
   - Each order_item references exactly one product
   - Foreign key: order_items.product_id → products.product_id

4. CUSTOMERS → USER_SESSIONS (One-to-Many)
   - One customer can have many sessions
   - Each session belongs to exactly one customer
   - Foreign key: user_sessions.customer_id → customers.customer_id

================================================================================
                              NOTATION KEY
================================================================================
   PK  = Primary Key (uniquely identifies each row)
   FK  = Foreign Key (references another table's primary key)
   1:N = One-to-Many relationship
   ►   = References/points to
